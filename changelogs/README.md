# 变更日志目录

本目录专门用于存储项目代码变更的摘要记录。

## 用途

每次对代码库进行重大修改时，应在本目录中创建一个摘要文档，记录以下内容：
- 修改了什么
- 为什么进行修改
- 修改对系统的影响
- 其他与修改相关的重要信息

## 命名方式

文件应使用以下格式命名：`YYYY-MM-DD-文档内容汇总.md`

例如：`2025-11-14-文档内容汇总.md`

## 文件格式

每个变更日志文件应包含以下内容：

```markdown
# 变更摘要 - [简要标题]

## 日期
YYYY-MM-DD

## 作者
[作者姓名]

## 描述
[详细描述所做的更改]

## 修改的文件
- file1.go
- file2.go
- etc.

## 影响
[对系统影响的描述]

## 备注
[其他补充说明]
```
